# Userbot

Подробная инструкция по установке, запуску и безопасной работе с этим Python-проектом (userbot).

## Описание

Этот репозиторий содержит простой userbot на Python. Файлы:
- `main.py` — точка входа бота.
- `prompt.py` — вспомогательные утилиты/набор подсказок.
- `session_name.session`, `session_name.session-journal` — локальные файлы сессии (не включать в репозиторий).

## Требования

- Python 3.10 или новее
- Windows (инструкции ниже для `cmd.exe`, но шаги аналогичны для PowerShell/Unix)
- Сеть для установки зависимостей

Зависимости перечислены в `requirements.txt`.

## Быстрая установка (Windows, cmd.exe)

1) Откройте командную строку и перейдите в папку проекта:

```cmd
cd C:\Users\Laziz\Desktop\userbot
```

2) Создайте виртуальное окружение и активируйте его:

```cmd
python -m venv .venv
.\.venv\Scripts\activate
```

Если используете PowerShell: `.\.venv\Scripts\Activate.ps1`.

3) Установите зависимости:

```cmd
pip install -r requirements.txt
```

4) Настройте переменные окружения (рекомендуется через `.env` файл). Пример `.env`:

```env
# Пример .env — НЕ коммитить реальные значения
API_ID=123456
API_HASH=your_api_hash_here
BOT_TOKEN=123456:ABC-DEF
SESSION_NAME=session_name
```

Создайте файл `.env` в корне проекта и заполните свои значения. Файл `.env` уже исключён в `.gitignore`.

## Запуск

Чтобы запустить проект:

```cmd
cd C:\Users\Laziz\Desktop\userbot
.\.venv\Scripts\activate
python main.py
```

Если `main.py` ожидает аргументы или конфигурацию — проверьте документацию внутри файла или комментарии в коде. Я могу помочь адаптировать `README` под реальный интерфейс запуска, если покажете `main.py`.

## Безопасность и работа с сессиями

- Файлы `*.session` и `*.session-journal` содержат токены/сессии — не публикуйте их в репозитории.
- Если случайно закоммитили сессию, удалите её из индекса и истории:

Удалить из индекса (оставив локально):

```cmd
git rm --cached session_name.session
git rm --cached session_name.session-journal
git commit -m "Remove local session files from repo"
```

Если сессии и секреты попали в историю, используйте BFG или `git filter-repo` (могу дать точные команды по запросу).


Полезно: перед пушем убедитесь, что сессионные/секретные файлы не попали в коммит (см. `git status`).

## Отладка и часто встречающиеся проблемы

- SyntaxError при запуске: запустите `python -m py_compile main.py prompt.py` — покажет синтаксические ошибки.
- Не хватает зависимостей: активируйте виртуальное окружение и `pip install -r requirements.txt`.
- Проблемы с виртуальным окружением: удалите `.venv` и создайте заново, убедитесь, что используете правильную версию Python.
- Если бот не подключается — проверьте правильность `API_ID`, `API_HASH`, `BOT_TOKEN`.

## Разработка

Рекомендации для локальной разработки:
- Используйте виртуальное окружение (`.venv`).
- Добавьте `requirements-dev.txt` для dev-зависимостей (linters, тесты).
- Запустите линтер (например, `pyflakes` или `flake8`) перед коммитом.
